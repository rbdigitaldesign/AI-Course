Large Language Models

Words

The first task of any language-based model is to encode words into numbers. Computers think in numbers after all. The question then is, how to do this? A naive approach might just be to list all of the words in the dictionary and give each a number from 1 to... 100,000 or so? Well, this isn't ideal. We want the mathematics of the numbers to reflect the words meaning in a useful way. If cat is 5 and dog is 7, does that mean that "cat + dog" is 12 (which could be pumpkin?).

A second approach might be to make all the words independent. For example we could create an array of boxes. When you want to represent a word, you simply put in '1' in the box you are interested in and a '0' everywhere else (Figure 1).

 

cat dog mouse ... ... ...
0 1 0 0 0 ...

Figure 1: Example of an array of numbers for each word. Dog is one... everything else is zero.

This isn't particularly satisfying either. If we present the data this way, every word is independent of every other word. For example, cat and dog have nothing in common (Figure 2). You could be more 'catty' or less 'doggy'. One might wonder what the point [1, 1] represents on this graph (a half-cat-dog)?

So we need a compromise. We need the numbers to not be completely independent (words are connected in some ways), but not dependent in silly ways (cat + dog = pumpkin). Ideally, we would want a system where the dimensions have some kind of abstract meaning, which might allow us to add words together in a meaningful way. For example, we might think of the king-queen relationship being similar to the churl-wench relationship (for reference, a churl is a male peasant and a wench is female peasant). We might imagine:

  • man + royal = king
  • woman - royal = peasant
  • OR even (this might be a stretch)... king - churl + wench = queen

Let's assume for the moment, we can take all the words in existence that have ever been uploaded to the internet, put them into a magical deep learning box and get it to learn the relationships between all words. One way we can set this up is to simply try to predict missing words in sentences.

  • That is the ______ sandwich I have ever eaten.

One can clearly see that some words (best, worst) fit here quite well, but others don't. We can reward the model for getting words that should appear here, and punish it for words that shouldn't appear here and eventually (if we train the model for long enough) we should get a dictionary of words, represented by arrows pointing in all different directions in thousands of dimensions (yes... we need more than 3 dimensions... yes... that is hard to imagine).

Sentences

Even if we have definitions for words, that isn't enough. Context matters. Consider:

  • Time flies like an arrow.
  • Fruit flies like an orange.

While superficially similar, the way your brain interprets these two sentences is completely different.
